# README

## Overview

This repository contains the **analysis artifacts and experimental results** used in our study. The results are derived from running the measurement frameworks described in the paper.

The directory includes analysis for two components:

- Network Request Analysis of wallet extensions
- Web Exposure Analysis of wallet provider interactions

## Network Request Interceptor

We provide analysis results for three wallet datasets:

- cws-10k-85
- torres-2023-100
- torres-2025-100

Each dataset directory contains:

```
dataset/
├── analysis_result             # Results produced by run_analysis.sh
├── data                        # Raw measurement data collected during experiments
├── extensions.csv              # Metadata about the analyzed extensions
├── extensions.txt              # List of extension IDs included in the dataset
└── wallet-info.csv             # Information collected during the experiment
```

The same structure is used for all datasets:

- cws-10k-85
- torres-2023-100
- torres-2025-100

```bash
cd analysis
docker build -t analysis .

# Generate Table 1
docker run --rm -it \
  -v "$(pwd):/work/analysis" \
  analysis \
  -lc "cd request-interceptor && ./run_experiment1.sh"

# Generate csv of Table 2
docker run --rm -it \
  -v "$(pwd):/work/analysis" \
  analysis \
  -lc "cd request-interceptor && ./run_analysis.sh ./cws-10k-85"

# Generate Table 4
docker run --rm -it \
  -v "$(pwd):/work/analysis" \
  analysis \
  -lc "cd web-exposure-analysis/2\)-30-dApps-behavior && python3 dapps-analysis.py && python3 generate_table4.py"

```

Please refer to `../ARTIFACT-APPENDIX.md` for details.

### Analysis Outputs

The request interceptor analysis produces several measurements in `./<dataset>/analysis-result/`

1. Main result #1: Number of Wallets Leak Address
   File `./analysis_result/leaks_type/ulti-merge.csv` reports:
   - wallets leaking addresses to analytics endpoints (Column "analytics?")
   - wallets leaking addresses to telemetry endpoints (Column "telemetry?")
   - total wallets leaking addresses (Add all columns together)

2. Main result #2: Third-Party Domains Receiving Addresses
   File `./analysis_result/tracker_presence/wallet_leak_domains.csv` reports:
   - The total number of domains receiving addresses (row count)
   - domains contacted by only one wallet (unique_extensions_with_wallet_leak = 1)
   - the most frequently contacted domains (top 5 ranked by "unique_extensions_with_wallet_leak")

3. Main result #3: Analytics presence in wallets
   File `./analysis_result/tracker_presence/analytics_summary.csv` reports:
   - wallets embedding analytics/telemetry endpoints ("analytics_hits" >= 1)

## Web Exposure

The Web Exposure framework collects a `.json` file for each experiment containing:

- `cookieAndLocalStorage`: changes to cookies and localStorage
- `objectEventLogs`: interactions with the wallet provider object
- `pageRequests`: network requests generated by the dApp
- Some other metadata

In our datasets, we show these results from our experiments:

1. Wallets revocation behavior (Web Threat #2 #3)
   - `1)-cws-10k-85-revocation`: Raw results from testing whether wallets implement the `wallet_revokePermission` method.
   - `1)-cws-10k-85-revocation-summary.csv`: Manually aggregated summary of the results.
2. dApps behavior
   - `2)-30-dApps-behavior`: what a dApp accesses a wallet provider object, and what a dApp sends out to the externals
     - `results`: raw experiment outputs
     - `dapps-analysis.py`: to analyze the `results`. (run: `python dapps-analysis.py`)
   - `2)-dApps-behavior-summary`: output of `dapps-analysis.py`
     - Do not invoke `wallet_revokePermissions` ("revokePermission" == False)
     - Calls `eth_accounts` before permission: ("eth_accounts_beforeRequest" == True)
     - Stores address in localStorage/cookies: ("localStorage" == True)
     - Do not clear stored address on logout: ("localStorage" == True AND walletStoredAtEnd == True)
     - Contacts >= 1 third-party trackers: ("tracker_count" >= 1)
     - Contacts >= 3 third-party trackers: ("tracker_count" >=3)

3. Wallets iframe injection check (Web Threat #4)
   - `3)-cws-10k-85-iframe-injection-summary`, manually collected results from experiments testing whether wallets inject their provider objects into cross-origin iframe contexts.
