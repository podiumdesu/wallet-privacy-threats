Threat #4 Demo – “Tracker” Website

This page simulates a normal Web2 site (e.g., websiteA.com) that embeds a dApp on another origin. A shared third-party script (“tracker”) runs here and inside the dApp. The tracker can call eth_accounts and learn your wallet address if the wallet is still connected to that origin.

Step 1
Connect your wallet to the demo dApp
Open the demo dApp, connect your wallet once, then close the tab. You won’t interact with this tracker page at all.
Step 2
See what this tracker page can read
Come back here. The tracker will query eth_accounts and show which wallet addresses are silently exposed to this origin.

Tracker’s view of wallet exposure inside the dApp (for demonstration, the iframe is visible here but an attacker could hide it completely):

The same tracker script runs inside this iframe and may call eth_accounts from the dApp origin.

2 What this page can read from your wallet

Without any obvious wallet UI here, the tracker script on this origin (websiteA.com) runs eth_accounts. If your wallet is still connected to this origin, your address will appear below. This is what we want to prevent with proper revocation.

Use this after revoking permissions in your wallet to see the effect.
Waiting for results…
As soon as the tracker runs, you will see entries for:
• top-level — what this Web2 site can read directly.
• iframe — what the dApp origin can read and send back via postMessage.
